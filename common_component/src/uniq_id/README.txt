uniq_id是一个每秒可产生千万级唯一ID的服务
弱主从架构，具备高可用性，包含master在内的任意节点挂掉，均不影响服务。
基于租约的思想，架构和实现均十分简单。

应用场景：
1）8字节整数的唯一ID，可用于订单号等
2）各类包含日期和业务类型等的订单号、流水号、交易号等，如

如何保证ID的唯一性？
1）为每台机器分配唯一的Label（标签），uniq_id的实现支持Lable取值1~255，也就是最多254台机器
2）每台机器自维护一个4字节无符号的循环递增Sequence（序号）
3）唯一ID加上日期时间（年、月、日、小时等）
4）以上3部分组成即可保证唯一性。

使用租约管理Label，以简化Label的维护，租期可定制，但建议以天为单位。
uniq_id各组成均为单进程无线程，成员间通讯采用UDP协议，以致实现轻巧但又高效。
